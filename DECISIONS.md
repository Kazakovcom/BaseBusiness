# Ключевые решения этапа 1

1. **Laravel + Blade для первого этапа**
   - Выбран минимальный серверный baseline без SPA и сложного фронтенда.

2. **SQLite для локального baseline-этапа**
   - Минимальный порог запуска и простая локальная конфигурация для тестового задания.

3. **Упрощённая авторизация через выбор пользователя**
   - Позволяет проверить ролевой каркас без внедрения регистрации и паролей на этом этапе.

4. **Доменная модель заявки названа `ServiceRequest`**
   - Избегается конфликт и путаница с `Illuminate\Http\Request`.

5. **Публичный экран создания заявки**
   - `GET /requests/create` и `POST /requests` доступны без role-check middleware, как требуется на baseline-этапе.

6. **Поэтапная реализация без преждевременной бизнес-логики**
   - На этом шаге есть только каркас и создание заявки; операции назначения/взятия/завершения намеренно не реализованы.

7. **Docker startup с проверками наличия `.env`, `sqlite` и `vendor`**
   - Для локального запуска в одном контейнере добавлен предсказуемый стартовый скрипт с миграциями и сидами.

> Примечание: формулировки в этом файле описывают принятые решения и состав кода, но не являются заявлением о подтверждённом запуске в текущем окружении агента.
