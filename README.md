# BaseBusiness

Тестовое задание "Заявки в ремонтную службу" на Laravel. В проекте реализованы создание заявки, панель диспетчера, панель мастера и защита конкурентного сценария "Взять в работу".

## Стек

- PHP 8.3
- Laravel 12
- Blade
- SQLite
- Docker Compose
- PHPUnit
- Bash + curl для ручной проверки race condition

## Локальный запуск

Предпочтительный и описанный в проекте способ запуска: через Docker Compose.

```bash
docker compose up --build
```

После старта контейнер `app` автоматически:

1. создаёт `.env` из `.env.example`, если файла ещё нет;
2. создаёт `database/database.sqlite`, если файла ещё нет;
3. выполняет `composer install`, если отсутствует `vendor/autoload.php`;
4. генерирует `APP_KEY`, если ключ пустой;
5. выполняет `php artisan migrate --seed --force`;
6. запускает приложение на `http://localhost:8000`.

Если нужно повторно применить миграции и сиды вручную в уже запущенном контейнере:

```bash
docker compose exec app php artisan migrate --seed --force
```

## Тестовые пользователи

Сиды создают трёх пользователей:

- `dispatcher@example.com` — диспетчер
- `master1@example.com` — мастер
- `master2@example.com` — мастер

Авторизация упрощённая: вход выполняется через страницу `/login` выбором пользователя из списка. Пароли не используются.

## Основные страницы

- `/` — главная страница с кратким описанием
- `/login` — вход по выбору пользователя
- `/requests/create` — создание новой заявки
- `/dispatcher` — панель диспетчера
- `/master` — панель мастера

## Что умеет диспетчер

- видеть список всех заявок;
- фильтровать список по статусу;
- назначать мастера только для заявки в статусе `new`;
- отменять заявку только в статусах `new` и `assigned`.

После назначения заявка переходит в `assigned`. После отмены заявка переходит в `canceled`, а `assigned_to` очищается.

## Что умеет мастер

- видеть только заявки, назначенные на него;
- брать в работу только свою заявку в статусе `assigned`;
- завершать только свою заявку в статусе `in_progress`.

Поддерживаемые переходы статусов со стороны мастера:

- `assigned -> in_progress`
- `in_progress -> done`

## Защита race condition для take in work

Операция "Взять в работу" реализована не через предварительную проверку статуса в PHP, а через атомарный условный `update`:

- обновление выполняется только для нужного `id`;
- только если `assigned_to` совпадает с текущим мастером;
- только если текущий статус равен `assigned`.

Это означает, что при двух параллельных запросах к одному и тому же `assigned`-запросу только один запрос сможет перевести заявку в `in_progress`. Повторный или проигравший запрос получает контролируемый отказ и не делает второй успешный переход.

## Как вручную проверить гонку

Вариант через браузер и DevTools:

1. Запустите проект и войдите как `master1@example.com`.
2. Убедитесь, что у мастера есть заявка в статусе "Назначена". На чистой базе после `migrate --seed` такая заявка создаётся сидером.
3. Откройте DevTools и на вкладке Network найдите POST-запрос формата `/master/requests/{id}/take`.
4. Скопируйте запрос как cURL два раза.
5. Запустите оба cURL почти одновременно в двух терминалах.
6. Ожидаемый результат: один ответ успешный, второй получает контролируемый отказ, а статус заявки меняется только один раз.

Более удобный вариант автоматизации описан ниже через `race_test.sh`.

## Как запустить `race_test.sh`

Скрипт находится в корне репозитория и предназначен для проверки именно конкурентного сценария `take`.

Базовый запуск:

```bash
bash ./race_test.sh
```

Поддерживаемые переменные окружения:

- `BASE_URL` — адрес приложения, по умолчанию `http://localhost:8000`
- `MASTER_USER_ID` — `id` мастера для логина, по умолчанию `2`
- `REQUEST_ID` — `id` заявки; если задан, скрипт сразу использует его для двух `take`-запросов, а если не задан, сначала открывает `/master` и ищет первую доступную кнопку `take`

Пример:

```bash
BASE_URL=http://localhost:8000 MASTER_USER_ID=2 bash ./race_test.sh
```

Скрипт:

- получает CSRF-токен со страницы логина;
- логинится в сессию мастера;
- использует токен со страницы логина для POST-запросов и не требует повторного извлечения CSRF с `/master`;
- при переданном `REQUEST_ID` сразу проверяет нужную заявку;
- при пустом `REQUEST_ID` открывает `/master` только для автопоиска доступной assigned-заявки;
- отправляет 2 параллельных POST-запроса на `/master/requests/{id}/take` с `Accept: application/json`;
- печатает HTTP-коды и тела обоих ответов;
- завершает выполнение ошибкой, если не получилось получить ровно один успешный результат и один контролируемый конфликт;
- выводит отдельные диагностические ошибки для проблем логина, неверной роли, отсутствия assigned-заявки и неожиданных HTTP-кодов.

Если на текущей базе у выбранного мастера нет заявки в статусе `assigned`, сначала повторно примените сиды или передайте подходящий `REQUEST_ID`.

## Автотесты

Запуск тестов из контейнера:

```bash
docker compose exec app php artisan test
```

Альтернативно через Composer внутри контейнера:

```bash
docker compose exec app composer test
```

## Какие тесты уже есть

- `tests/Feature/HomePageTest.php` — доступность главной страницы
- `tests/Feature/ServiceRequestCreationTest.php` — создание заявки со статусом `new`
- `tests/Feature/DispatcherPanelTest.php` — список заявок диспетчера, фильтр по статусу, назначение мастера, отмена заявки, сохранение фильтра после POST и негативные сценарии диспетчера
- `tests/Feature/MasterPanelTest.php` — список только своих заявок, успешные `take` и `complete`, JSON-конфликт при повторном `take`, негативные сценарии мастера и защита маршрутов по роли
- `tests/Feature/MasterRequestWorkflowServiceTest.php` — повторный вызов `takeInWork()` для одной и той же заявки: первый успешен, второй получает контролируемый отказ без повторного изменения данных

## Ограничения и допущения

- Авторизация упрощённая: вход без пароля, только выбором пользователя из сидов.
- По проекту описан и поддерживается Docker-способ запуска; отдельный без-Docker сценарий в README не документируется.
- Для ручной проверки race condition нужен мастер с заявкой в статусе `assigned`. На чистой базе это обеспечивается сидерами.
